System Overview

Re.Match is a constraint-based opportunity matching engine for justice-involved individuals. It maps personal constraints (debt, supervision, addiction) against all available resources to produce god-tier recommendations — maximum life impact, 100% eligibility match.

3. Research & Matching Workflow

Step 1: Intake → Profile

Form → JSON profile (80 fields max). Audio transcription optional.

{
  "constraints": ["felony", "debt_5k", "probation"],
  "interests": ["tech", "fitness", "music"],
  "special_status": ["veteran", "dv_survivor"]
}

Step 2: Database Query + No-Redundancy

Query opportunity DB first. Only research NEW items.

Score formula: (Eligibility × 1.0) + (Life Impact × 1.0) + (Speed × 0.5)

Step 3: God-Tier Selection

  • Top 1 per category → PDF
  • Top 5 per category → Full DB link
  • Action plans ordered by dependency

Action Plan Logic

  1. Long-wait, no-prereq (housing apps)
  2. Must-haves (ID/docs)
  3. Income + stability
  4. Growth (education/career)

6. Technical Implementation

Database Schema (SQL)

CREATE TABLE opportunities (
  id INTEGER PRIMARY KEY,
  category_id INTEGER,
  name TEXT,
  eligibility_criteria JSON,
  life_impact_score REAL,
  speed_days INTEGER,
  contact JSON,
  location TEXT,
  status TEXT DEFAULT 'active'
);

CREATE TABLE profiles (
  id TEXT PRIMARY KEY,
  intake_json JSON,
  constraints JSON,
  matches JSON
);

Low Literacy Design Rules

  • Max 15 words/sentence
  • 1 action per box
  • Big headings, short paragraphs
  • Phone/address first
  • Colored priority badges

7. Dossier Generation Workflow

  1. Intake form → JSON profile
  2. Query DB with no-redundancy filter
  3. Research gaps (web + local scrape)
  4. Score & rank (eligibility × impact)
  5. Build master action plan (dependency sort)
  6. Generate HTML → PDF
  7. Host full DB link

8. Final Doctrine Notes

Key Principles

  • Action-first: Steps before explanations.
  • No redundancy: DB prevents repeat research.
  • Jackpot-first: Special status unlocks 10x resources.
  • RNR-aligned: Every category maps LS/CMI domains.
  • Scalable: One profile → infinite dossiers.

Success Metrics

  • 100% eligibility match
  • Life impact > 90%
  • Action completion rate > 70%
  • Recidivism reduction proxy: stability metrics
