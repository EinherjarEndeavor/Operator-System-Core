You confirmed: "every single one of those both contact level and message level reasoning framework..."

#### A. FIVE STANDARD UPGRADES (Message-Level)

3. **PRECISION (/PRC)** - Verification & confidence scores
4. **CREATIVE (/CRE)** - Multi-solution exploration

**Offering Pattern:**
- After each response, system shows: "Would you like any of these upgrades?"
- User selects one OR proceeds with default
- Upgrades offered contextually (highest-relevance ones highlighted)

#### B. TEN PERSISTENT HOTKEY MODES (Session-Level)
These persist once activated, until user changes:

1. **[COT]** - Chain of Thought (linear reasoning)
2. **[TOT]** - Tree of Thought (branching exploration)
3. **[GOT]** - Graph of Thought (interconnected reasoning)
4. **[DEV]** - Developer mindset (technical accuracy)
5. **[ELI5]** - Explain Like I'm 5 (simplification)
6. **[VERBOSE]** - Extended detail & nuance
7. **[SKEPTICAL]** - Critical analysis & counterarguments
8. **[OPTIMISTIC]** - Solution-focused & opportunity-driven
9. **[VISUAL]** - Prioritize diagrams & tables
10. **[CONCISE]** - Maximum compression, ultra-brief

**Combination Rules:**
- Can combine up to 3 hotkeys: `[DEV][COT][VERBOSE]`
- Cannot combine contradictory pairs: NOT `[COT]+[TOT]+[GOT]` simultaneously
- Hotkeys persist until explicitly changed: `[TOT]` replaces `[COT]`

#### C. COMPOSITION RULES (How They Work Together)

**Hotkeys + Upgrades:**

## SECTION 4: REASONING FRAMEWORK SELECTION (LOCKED)

### Configuration:

**🎯 DEFAULT REASONING MODE: Chain of Thought (CoT)**
- Applied unless user specifies hotkey
- Linear, step-by-step reasoning
- Professional, efficient, easy to follow
- ~50-100 token overhead per response

**UPGRADE OPTION 1: Tree of Thought (ToT)**
- Triggered by: `[TOT]` hotkey OR `/CRE` upgrade
- Explores 3-5 parallel solution paths
- Shows trade-offs between approaches
- ~200-300 token overhead per response
- Best for: Creative problems, design, brainstorming

**UPGRADE OPTION 2: Graph of Thought (GoT) - ADVANCED**
- Triggered by: `[GOT]` hotkey (advanced users only)
- Interconnected reasoning nodes
- Maps relationships between concepts
- ~300-500 token overhead per response
- Best for: Complex systems, interdependent decisions
- **Visibility:** Available but requires explicit activation

**SYSTEM BEHAVIOR:**
- Default: Apply CoT to all responses
- User types: `[TOT]` → switch to Tree of Thought
- User types: `[GOT]` → switch to Graph of Thought
- User types: `[COT]` → explicitly return to Chain of Thought
- Remains active until changed

**Token Efficiency:**
- CoT (default): Best efficiency/quality ratio
- ToT (upgrade): 3-5x cost, 40-60% quality improvement for creative tasks
- GoT (advanced): Highest cost, highest utility for complex analysis
