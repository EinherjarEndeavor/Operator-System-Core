  1. The "MapReduce" Agentic Workflow (For Exhaustiveness)
  When a folder is too big for a single context window, standard RAG (Retrieval-Augmented Generation) fails because it only looks at "relevant snippets" (top-K). For a "tell me everything" request, you need
  MapReduce:
   * The Map Phase: A fleet of "Scout Agents" (or multiple calls to a single agent) is assigned to specific chunks of the folder. Their instruction is: "Extract EVERY instance of [Concept/Tool/Logic]. Do not

   * The Reduce Phase: A "Synthesis Agent" takes one category at a time (e.g., "All Network Security Tools") and writes the definitive, expanded section for that category.
   * Tools: LangGraph or CrewAI are perfect for this. They allow you to maintain "state" across 50 turns without losing the thread.

  2. "Axiom-Mechanism-Tension" Extraction (For Abstract Concepts)
  If you want to extract "The philosophy of the system" rather than just "Tool names," you cannot use simple keyword searches. You use Theoretical Frameworks as your prompt's "Lenses":
   * Instruction: "Analyze these documents through the lens of 'The MegaMan X Metaphor.' For every file, identify the Axioms (underlying truths), the Mechanisms (how those truths are enforced), and the
     Tensions (where the system fails or conflicts)."
   * This forces the LLM to generate new, high-level analysis instead of just repeating what it sees.

  3. "Recursive Expansion" (The Outline-to-Deep-Dive Method)
  To get a 100-page report from a 10-page folder, you use a recursive loop:
   * Step 1: Generate a 200-point granular outline of the entire folder.
   * Step 2: For each point in the outline, create a new prompt: "Using these 3 source files as reference, write a 500-word deep dive on point #47. If the source file mentions a tool, look up its documentation
     via web search to provide a full implementation guide."
   * Management: This requires a State Machine (like a Python script) that tracks which points are finished and which are pending.

  4. Knowledge Graph Construction (The "No-Loss" Format)
  Instead of a Markdown file, the "final result" of your onboarding should be a Knowledge Graph (using Neo4j or a simple JSON-LD structure).
   * Why? A text document is linear and always loses information. A graph records every relationship: [Tool A] -> [Used For] -> [Research Task B] and [Research Task B] -> [Requires] -> [Axiom C].
   * Libraries: GraphRAG (by Microsoft) is a new library specifically designed for this. It uses LLMs to build a graph of an entire folder, allowing you to ask "Global Questions" that are much more accurate
