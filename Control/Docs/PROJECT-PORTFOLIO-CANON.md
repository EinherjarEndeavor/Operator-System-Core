# PROJECT PORTFOLIO CANON

Status: Proposed canon
Purpose: Collapse the major initiatives scattered across Wave1Canon and related piles into a single operational portfolio map.

## 1. Portfolio summary

The current workspace revolves around five primary initiatives:

1. **RVE (Rolling Victory Engine / Revival Vector Engine)**
   - Role: personal operating system for daily execution, constraints, habits, scheduling, and context export.
   - Core value: convert chaos into structured execution.
   - Current bottleneck: truth routing and reliable task-state operation.

2. **Re.Match**
   - Role: constraint-based opportunity matching and dossier generation for justice-involved / recovery / housing-insecure people.
   - Core value: connect a complete profile to live, relevant resources and next-step action plans.
   - Current bottleneck: canonizing service definition, intake fields, matching rules, and dossier production flow.

3. **DeepRick / Mycroft Holmes / Deep Research Engine**
   - Role: research and synthesis engine for turning scattered source piles into high-grade reports and reusable knowledge.
   - Core value: transform fragmented notes and reports into coherent, decision-grade doctrine.
   - Current bottleneck: source curation, canon workflows, and avoiding slop/compression.

4. **Second Wind**
   - Role: broader recovery / reentry / self-transformation endeavor.
   - Core value: translate the user's lived reconstruction into tools, philosophy, and support systems for others.
   - Current bottleneck: clear boundary between mission statement, doctrine, and executable product lines.

5. **Content / Online Endeavors**
   - Role: turn knowledge, systems, and lived experience into portfolio assets, public-facing work, and income paths.
   - Core value: convert accumulated knowledge into outputs people can consume, trust, and pay for.
   - Current bottleneck: scattered positioning, no unified campaign map, too many parallel ideas.

## 5. Current priority stack

### Priority 1
RVE operational MVP
Reason: if the personal operating system is not usable, everything else is harder to execute.

### Priority 2
Command/workflow canon + truth routing
Reason: prevents continual re-derivation and canon corruption.

### Priority 3
Re.Match feature canon
Reason: highest external usefulness and strongest path to immediate service value.

### Priority 4
Deep research canonization pipeline
Reason: unlocks the ability to transform the massive corpus into reusable assets.

### Priority 5
Public-facing battle plan
Reason: convert capability into visible artifacts and opportunities.

## 6. What to stop doing

- Treating all files as equal authority
- Mixing mission prose with operational truth
- Designing all systems simultaneously
- Expanding infrastructure before current products have clear operational contracts

## 7. Immediate build principle

Each project should now move from scattered brainstorming into:
1. one canon overview
2. one feature-driven spec
3. one current-state / next-state map
4. one active execution list in RVE

That is the portfolio discipline required to stop the workspace from collapsing back into slop.
