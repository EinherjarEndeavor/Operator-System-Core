# WAVE1CANON CANONIZATION PIPELINE

Status: Proposed canon process doc
Purpose: Define how messy Wave1Canon material gets converted into stable, integratable sources of truth instead of remaining a scattered file swamp.

## 1. Objective

Wave1Canon and adjacent piles contain useful material, but usefulness is currently trapped inside duplication, scattered framing, and mixed authority.

The goal is not to preserve every file as equally sacred.
The goal is to refine the corpus into:
- truth records
- feature specs
- workflow canon
- artifact canon
- deferred/archive material

## 2. Canonization classes

### Class A — Truth source
Use for direct, structured facts and current operating constraints.
Examples:
- profile/intake answers
- direct user clarifications
- stable obligations and logistics

### Class B — Feature / design source
Use for product definitions, schema ideas, and workflow mechanics.
Examples:
- RVE master briefing
- memory system proposition
- slash/workflow raw files
- Re.Match service description

### Class C — Artifact / voice source
Use for narrative, persuasive, and presentational assets.
Examples:
- cover letters
- resume prose
- mission statements
- longform doctrine artifacts

### Class D — Redundant / deferred source
Use for duplicate passovers, repeated brainstorms, and low-priority side piles.
These are preserved but not used as primary authority.

## 3. Conversion outputs

Every messy source should be converted into one of the following output types:

1. **Routing proposal**
   - what layer(s) it belongs to
   - what should be extracted
   - what should remain artifact-only

2. **Canon spec**
   - coherent feature-driven source of truth for a project or subsystem

3. **Battle plan / action plan**
   - executable direction and sequencing

4. **Deferred archive note**
   - why the source is not currently primary and when to revisit it

## 4. Minimal canonization workflow

1. identify the source class
2. identify whether it improves truth, tasks, workflows, artifacts, or not-now archives
3. extract only the highest-authority content
4. convert into a single actionable canon doc
5. record what remains deferred or unresolved

## 5. What not to do

- do not merge everything into one giant mega-doc
- do not preserve rhetorical duplication as if it were new information
- do not treat brainstorming and canon as the same class
- do not let narrative style outrank operational truth

## 7. Success condition

Wave1Canon is no longer the place you must think from.
It becomes the ore reserve.
The Control/Docs canon becomes the place you actually operate from.

## 8. Red-team warning

This process fails if it becomes another endless summarization ritual.
It only works if each canon document replaces scattered dependence with a sharper operational reference.
