# MINIMAL NEXT STEPS PLAN

Status: Proposed execution plan
Purpose: Prevent overbuilding and define the smallest set of moves that will create real utility.

## First principle

Do not try to finish the entire grand architecture right now.
The current phase is successful if it produces:
- trustworthy profile truth
- trustworthy task / obligation state
- trustworthy command and workflow canon
- a repeatable ingestion path

## What is actually worth doing right now

### Worth doing now
1. command / workflow canon
2. truth routing rules
3. selective Layer 2 ingestion
4. selective Layer 4 artifact preservation
5. task ingestion into the operational database

### Not worth prioritizing right now
1. new memory layers
2. advanced graph work
3. large semantic infrastructure
4. speculative automation sprawl
5. perfect elegance before basic truth and task retrieval work

## Recommended sequence

### Step 1
Finish command and workflow canon.
Outcome: cheap, reusable cognition surface.

### Step 2
Use truth-routing rules to ingest only direct structured facts from the high-value source set.
Outcome: clean profile ground truth.

### Step 3
Ingest task-heavy material into the operational task system.
Outcome: the system starts helping with real execution, not just self-description.

### Step 4
Store resumes, letters, certificates, and doctrine in Layer 4 with selective extraction.
Outcome: useful artifact library without corrupting truth.

### Step 5
Only after the above, reassess whether advanced semantic or graph infrastructure is still the bottleneck.
Outcome: architecture grows in response to actual need.

## Reality check

The user is not wasting time by building a reusable operating system for cognition, execution, and recovery.
The user would be wasting time only if the work stayed permanently in the architecture and refinement phase without stabilizing truth, tasks, and retrieval.

## Success condition for this phase

This phase is successful when:
- direct facts can be retrieved reliably
- tasks / obligations are queryable
- command workflows are stable and reusable
- narrative artifacts are preserved without being mistaken for truth

## Failure condition for this phase

This phase is failing when:
- more layers keep getting added
- truth remains muddy
- no real task state exists
- every new file triggers a redesign spiral
