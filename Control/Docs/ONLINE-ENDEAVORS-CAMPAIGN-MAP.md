# ONLINE ENDEAVORS CAMPAIGN MAP

Status: Proposed hardening addendum
Parent document: `ONLINE-ENDEAVORS-BATTLE-PLAN.md`
Purpose: Convert the broad battle plan into audience-aware campaigns, flagship outputs, and anti-drift rules.

## 1. Core audiences

### Audience A — Programs / admissions / institutions
Need:
- evidence of seriousness
- evidence of follow-through
- coherent project explanations
- credible recovery + capability narrative

### Audience B — collaborators / engineers / builders
Need:
- clean repos
- feature-driven specs
- example artifacts
- clarity of what exists versus what is planned

### Audience C — constrained end-users / clients
Need:
- clarity
- usefulness
- direct next steps
- outputs that reduce overwhelm rather than increase it

### Audience D — opportunity providers / supporters / donors / partners
Need:
- mission clarity
- proof of utility
- believable scope
- evidence that the work helps real people

## 2. Flagship output types

The online layer should prioritize these flagship artifact classes:
1. project canon / README-quality overviews
2. case-study artifacts
3. service explainers
4. research-to-canon transformation examples
5. selected public-facing proof pages

## 3. Publish / private split

### Publish-ready by default
- project overviews
- feature-driven specs with sensitive details removed
- selected case studies
- general methodology docs
- public portfolio artifacts

### Private by default
- raw personal corpus
- sensitive constraints and legal specifics unless necessary
- internal control docs with operational security implications
- draft mission prose that is not yet sharp

## 4. Campaign priorities

### Campaign 1 — Proof of competence
Goal: show that the operator can turn chaos into clean deliverables.
Artifacts:
- portfolio canon
- RVE feature canon
- research-to-canon examples

### Campaign 2 — Service legitimacy
Goal: make Re.Match legible as a real service idea rather than a passionate monologue.
Artifacts:
- service explainer
- intake contract
- dossier structure

### Campaign 3 — Recovery-to-capability narrative
Goal: demonstrate that reconstruction and disciplined skill-building are real, not hypothetical.
Artifacts:
- concise mission pieces
- selected story fragments anchored in proof
- project outputs that embody the claim

### Campaign 4 — Opportunity capture
Goal: improve odds for admissions, partnerships, scholarships, and gigs.
Artifacts:
- tailored application materials
- proof-of-work docs
- focused project pages

## 5. Output cadence

Use a simple cadence:
- maintain canon internally first
- promote only sharpened artifacts outward
- prefer one strong flagship output over five scattered posts

## 6. Anti-drift rules

1. Do not publish architecture fantasies as if they are shipped systems.
2. Do not let branding outrun product clarity.
3. Do not create more channels than flagship artifacts.
4. Every public artifact should strengthen one of the core projects.
5. If a public-facing effort produces no proof, no opportunity, and no reuse, deprioritize it.

## 7. Success condition

The online layer is working when a stranger can understand:
- what the main projects are
- what is real today
- what problem each project solves
- why the operator is worth taking seriously
