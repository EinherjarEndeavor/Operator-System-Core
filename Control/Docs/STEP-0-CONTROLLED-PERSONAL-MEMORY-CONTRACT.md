# STEP 0 — CONTROLLED PERSONAL MEMORY CONTRACT

Status: ACTIVE
Purpose: Prevent the subjective-memory lane from becoming an indiscriminate personal-fact extraction engine. Define how personal material may be stored, queried, distilled, and promoted.

---

## 1. Executive judgment

Personal memory must not be auto-extracted by default.

The subjective-memory lane exists to support high-value queries, longitudinal analysis, self-observation, learning, planning, and opportunity discovery.

It is not a trivia collector.
It is not a therapist substitute.
It is not an always-on surveillance layer.
It is not a pipeline that turns every personal statement into permanent memory.

The correct model is a **controlled, mutually aware contract**.

The user must know when personal material is being logged, when it is being summarized, when it is being promoted, and what it will be used for.

---

## 2. Core rule

No personal subjective memory is promoted automatically.

Personal material may be stored or distilled only when one of the following is true:

1. the user explicitly invokes a journaling / reflection / review command
2. the user explicitly asks for analysis over personal logs, tasks, goals, projects, or trajectory
3. the user approves a proposed promotion from raw material into distilled pattern memory
4. the system is operating inside a clearly declared review workflow whose purpose is personal-memory analysis

Outside those contexts, personal statements remain conversation context only unless the user explicitly says otherwise.

---

## 3. What the lane is for

The subjective-memory lane should support queries such as:

- “Based on my logs, what trajectory changes should I make?”
- “What patterns appear across my task completion, projects, and goals?”
- “What conditions seem to support better learning?”
- “What repeated forms of friction are slowing me down?”
- “What opportunities are emerging from my interests, constraints, and recent work?”
- “What themes are visible in my writing or reflections?”

This lane is for meaningful pattern work and growth support.

---

## 4. What the lane is not for

The subjective-memory lane should not collect or preserve random personal details just because they appeared in conversation.

Examples of things that should not be auto-promoted:

- random preferences
- sexual details
- casual jokes
- one-off emotional remarks
- biographical trivia
- temporary venting
- unreviewed identity claims
- contextually irrelevant personal details

If such material matters, the user can explicitly place it into a journal, reflection, canon note, or controlled memory review.

---

## 5. Neutral-observation rule

The subjective-memory lane must start blind and unbiased.

It must not preload assumptions such as:
- addiction triggers
- relapse framing
- generic recovery stereotypes
- pathology-first interpretation
- clinical diagnosis
- weakness model

The user has explicitly clarified:
- do not assume urges
- do not assume triggers
- do not treat the user as a generic “person in recovery”
- do not overfit personal analysis to recovery stereotypes

Instead, the system should use neutral observational categories:
- state sample
- reflection
- skill-friction note
- opportunity signal
- pattern candidate
- project/life trajectory note
- learning condition note
- energy/capacity observation
- environmental constraint observation

---

## 6. Personal-memory storage classes

### Class A — Raw entries
Human-readable files such as journals, reflections, logs, and review notes.

Properties:
- explicitly created or imported
- append-mostly
- not treated as final truth
- queryable only when relevant and authorized

### Class B — Distilled observations
Structured summaries derived from raw entries or reviewed sessions.

Properties:
- source-linked
- confidence-scored
- revision-friendly
- not treated as identity truth

### Class C — Pattern candidates
Possible recurring patterns suggested by multiple observations.

Properties:
- explicitly marked as hypotheses or candidates
- must not be treated as diagnosis or fact
- should include evidence count and source references

### Class D — Matured pattern signals
Patterns that have been reviewed, repeated, and accepted as useful.

Properties:
- still not identical to technical truth
- usable by planning/coaching/review workflows
- subject to revision

---

## 7. Retrieval contract

Personal memory may be retrieved when:

1. the user asks for personal trajectory analysis
2. the user asks for review of journals/reflections/logs
3. the user invokes a subjective-memory or life-review command
4. a planning/review workflow explicitly declares that subjective-memory retrieval is relevant

Personal memory should not be silently injected into ordinary technical, coding, browser, or registry work.

---

## 8. Promotion contract

Before any personal material becomes a durable structured observation or pattern, the system should state:

- source material
- proposed memory class
- exact proposed summary
- why it is useful
- confidence level
- what it should affect
- what it should not affect

The user may approve, edit, reject, or defer.

---

## 9. Existing-system inspiration

Systems such as OpenClaw, Hermes, self-improving agents, ontology agents, and other memory-capable agent systems may be mined for useful mechanics.

The target is not blind imitation.
The target is to emulate useful patterns such as:
- explicit skill/memory contracts
- reviewable memory promotion
- separated raw logs and distilled memory
- auditability
- controlled retrieval
- modular capability surfaces
- self-improvement loops based on actual use

---

## 10. Relationship to subjective memory lane

This contract modifies the subjective-memory lane canon.

Any future `essence.db`, `essence_logs/`, subjective-memory skill, journal command, or psychological analysis engine must obey this contract.

The subjective-memory lane is allowed to become powerful.
It is not allowed to become indiscriminate.

---

## 11. Final judgment

The personal-memory system should be strategically saturated, not automatically saturated.

It should contain what is useful for growth, learning, planning, creative work, opportunity discovery, and self-directed analysis.

It should not become a pile of random personal facts.

The rule is:

**controlled capture, reviewable distillation, explicit retrieval, useful promotion.**
